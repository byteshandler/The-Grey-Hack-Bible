This glossary provides concise definitions for terms, concepts, tools, and terminology used throughout The Grey Hack Bible.

For detailed explanations and practical examples, refer to the relevant chapters.

| Term           | Definition                                                                            |
| -------------- | ------------------------------------------------------------------------------------- |
| Access         | The ability to interact with a system, account, service, or resource.                 |
| Account        | A user identity on a system that grants permissions and access.                       |
| Asset          | Any resource that provides value, such as servers, accounts, scripts, data, or money. |
| Automation     | The use of scripts and tools to perform tasks with minimal manual interaction.        |
| Backdoor       | A method of regaining access to a system after initial compromise.                    |
| Command        | An instruction executed through the terminal.                                         |
| Enumeration    | The process of gathering information about a target.                                  |
| Exploit        | A technique or tool used to take advantage of a vulnerability.                        |
| Filesystem     | The structure used to organize and store files and directories.                       |
| Greyscript     | The scripting language used within Grey Hack.                                         |
| Host           | A computer connected to a network.                                                    |
| Infrastructure | The collection of systems, servers, storage, and services supporting an operation.    |
| Methodology    | A repeatable process used to solve problems or achieve objectives.                    |
| Network        | A collection of interconnected hosts capable of communicating with one another.       |
| OPSEC          | Operational Security; the practice of protecting information, assets, and activities. |
| Persistence    | Techniques used to maintain access to a system over time.                             |
| Port           | A communication endpoint used by network services.                                    |
| Process        | A running instance of a program.                                                      |
| Reconnaissance | The process of identifying and gathering information about potential targets.         |
| Service        | A program that listens for and responds to network requests.                          |
| Target         | A host, service, account, or objective selected for investigation or action.          |
| Terminal       | A command-line interface used to interact with a system.                              |
| Tool           | A program designed to assist with a specific task.                                    |
| Trace          | Information that may reveal activity, identity, or actions performed on a system.     |
| Vulnerability  | A weakness that can be leveraged to gain access or perform unintended actions.        |
